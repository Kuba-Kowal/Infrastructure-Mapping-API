Passive DNS (pDNS) is a system that maps historical data of IP address - Domain mappings. This system doesn not generate DNS queries, rather monitors them being created. It is utilised to see
how infrastructure may have changed within an organisation, including revealing stale endpoints that are no longer maintained.
This data is captured by pDNS sensors (cameras inside of a library of DNS queries) that run on recursive resolvers (the place between authoritative name servers and caching servers) these systems
take clients queries and resolve them to IP addresses by accessing their own cachce or communicating with authoritative NSs.

It is possible to run your own pDNS sensor and recursive resolver, many people do for security, experimentation, learning DNS and filtering however the dataset for your pDNS will only be the 
requests that your client makes in-turn providing very little recon.

Considering many internal services will not be accessible publicly, and are resolved by internal name servers. pDNS provides a mapping of public facing infrastrucutre, however does not reveal
internal or private infrastructure.

Zone Files are publicly available text files that list all domains (not subdomains) mapped to a TLD. These provide you with a domain and their authoritative name servers, it can be useful for spotting
all domains under a TLD, related domains or typosquats.

-- Server Name Indication --
This is important when testing historical infrasturucture, specifically old CDN providers. it allows you to specify what hostname to reach behind a CDN.
example usage: curl --resolve domain:port:ip \
	 	https://hostname.comnmap

== Live DNS vs Historical DNS ==
Live DNS are current infrastructural mappings of Domain <-> IP address. These can be obtained utilising NSLOOKUP or dig.

Historical DNS are ones that been seen by pDNS sensors in the past, they do not necasserily map to live servers however can provide insite into infrastructural changes.
